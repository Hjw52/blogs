    深入CSS《二》格式化上下文

    Chuck 10/25/2020 CSS

    格式化上下文指的是元素所处的环境，在CSS中，每个渲染元素都有它对于的上下文。格式化上下文约束着元素在浏览器如何排列，如何显示及渲染等。通常格式化上下文分为：块格式化上下文（BFC）和行内格式化上下文（IFC）。要了解格式化上下文，我们先从清除浮动这个问题聊起。

    # 块级格式化上下文（BFC）

    下图为根元素构成的BFC，包含多个块。

    特点：

    1. 在一个BFC中，块框按垂直方向排列，相邻块外边距会重叠。（margin重叠问题）
    2. 盒子从包含块的最左边摆放，所有块级盒子的左边界默认与包含块的左边界对齐。
    3. BFC包含浮动的元素（这也是上面overflow:hidden；可以自适应浮动元素的原因）

    如何构成BFC：

    • 根元素本身就是BFC
    • 浮动float不为none
    • 脱离文档流的元素 position：absolute、fixed
    • display不为block
    • display为table-cell或table-caption
    • overflow不为visible的元素
    • 弹性盒子display为flex

    # 行内格式化上下文（IFC）

    特点：

    1. 在IFC中每个框都是一个接一个水平排列，起点为包含块的顶部，框的每一行称行框，行框的宽度由它的包含块及浮动元素决定，高度由line-height决定。
    2. 盒子一个接一个地水平摆放，容器宽度不够时换行（如不换行则溢出，如上图所示）
    3. 每一行生成匿名行盒，包含该行的所有行内级盒
    4. 当所有盒总宽度小于匿名行盒宽度时，水平方向的排版由 text-align 属性来决定，垂直方向上，行内级盒的对齐方式由 vertical-align 控制，默认对齐为 baseline基线对齐
    5. 行内盒垂直的border，padding和margin都不会撑开行盒的高度

    构成IFC：

    ​ 当块容器盒（block container box）不包括任何块级盒（block-level boxes）时，就会创建一个行内格式化上下文（IFC）。行盒的高度由其内容最高的一个决定。如果是都是纯 inline 元素，则其高度由最高的 line-height 计算值决定；而如果包括了 inline-block，则 inline-block 的 margin（上下），border（上下），padding（上下），height 都会影响整体行盒的高度。

    # 补充：行内元素对齐方式

    行内元素在盒子内垂直对齐的方式有以下基本线：

    • 基线：如图基线的位置，英文字母的下端，但有少许字母也会超过基线，如g。
    • 中线：英文字母的中间
    • 顶线：父元素内容区的顶部，与fontsize相关
    • 底线：父元素内容区的底部，与fontsize相关

    垂直对齐主要方式Vertical-align

    1. baseline：与父元素的基线对齐

    2. text-top：与父元素字体顶部对齐

    3. text-bottom：与父元素字体底部对齐

    4. middle：居中对齐，将元素中间与基线对齐加上父元素x-height的一半。（x-height为行高与内容区的高度差）
